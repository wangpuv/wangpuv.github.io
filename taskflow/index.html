Task Flow 项目的由来与简介

       2007年的一天，我偶尔结识了springside这个国内的开源项目，正是在阅读springside的同时，加深了我对spring的理解，并逐步走上了使用spring的道路。其实早在2005年，《程序员》杂志刊登“javaEE without EJB”的专题报道的时候，我就知道spring了，但是那个时候我刚毕业，正在学习java，而公司也还在使用EJB。时至今日，我已经熟练的使用spring框架开发项目了，同时，也在时刻关注着springside的成长。

       突然有一天，我萌发了使用springside框架开发一个开源项目的念头。恰巧，身边的朋友有任务管理系统的需求，于是利用空闲时间编写了一个很小的Demo——这就是TaskFlow。由于我的html+css+javascript基础比较差，前台页面只能摸索着前进，用了将近一个月的时间，搞定了这个demo的雏形。期间，也走了很多弯路，还有一些尚未解决的问题等待处理。

       Task Flow 是一个任务流转和管理的系统，其中也包含了机构设置与管理和任务查询、报表等内容。Task Flow 使用springside3 + hibernate3 + spring3 + struts2 进行开发，前台展现选择了JSP而非FreeMarker（不过struts2标签的模板使用的是FreeMarker）。其中包含了代码表、缓存、页面标签等一些技术特点，欢迎大家参考并指正其中的问题。

       TaskFlow使用Maven2进行管理和构建，这里说一下Maven。我是2007年学习Struts2的时候了解Maven的，立即产生了兴趣，但是直到现在Maven仍然没有进入我涉及的项目中，虽然绝大多数开源项目都已经在使用它了。所以，当我决定自己做一个开源项目的时候，Maven就成了必选。

       当然，编写 Task Flow 的目的还是希望能促进技术交流，也能间接促进springside项目的交流，我本人也很希望springside能健康的成长下去。同时，当然也希望这个任务管理系统还是能给需要的人或单位带来一些帮助，有这方面需求的可以发email给我。
